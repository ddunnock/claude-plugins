**Step 1 — Add `_extract_edges()` function:**
Implement per the research code example. Walk snapshot slots (interfaces, contracts, traceability-links) and build edge objects. Filter to only include edges where BOTH source_id and target_id are in the `in_view_ids` set. Sort edges for determinism by (source_id, target_id, relationship_type).

Edge relationship_type taxonomy:
- Interface with source_component -> edge from source to interface: `"component_interface"`
- Interface with target_component -> edge from interface to target: `"component_interface"`
- Contract with interface_id -> edge from interface to contract: `"interface_contract"`
- Contract with component_id -> edge from component to contract: `"component_contract"`
- Traceability-link -> edge from from_id to to_id: use the link's `link_type` field value (e.g., `"derives"`, `"satisfies"`)

**Step 2 — Add inline `relationships` field to each slot in sections:**
After sections are organized and ranked, for each slot in each section, add a `relationships` field listing connected slot IDs that are also in the view:
```python
slot["relationships"] = [edge["target_id"] for edge in edges if edge["source_id"] == slot["slot_id"]] + \
                        [edge["source_id"] for edge in edges if edge["target_id"] == slot["slot_id"]]
```
Deduplicate and sort the list. This is co-located data for human readability per user decision.

**Step 3 — Update `assemble_view()` to populate edges (replace empty placeholder):**
- Build `in_view_ids` set from all sections' slots' slot_id values
- Call `_extract_edges(snapshot, in_view_ids)`
- Replace the `"edges": []` placeholder with the actual edges list
- Add inline relationships to each slot (Step 2)

**Step 4 — Add structured logging throughout `assemble_view()`:**
Use the existing `logger` (line 24). All `extra` fields use `view.` prefix per user decision.

INFO log points (each with elapsed_ms):
1. After `capture_snapshot()`: "Snapshot captured: %d slot types, %d total slots" with extra `view.operation="snapshot_capture"`, `view.slot_count`, `view.elapsed_ms`
2. After each scope pattern match: "Section '%s': %d slots matched" with extra `view.operation="pattern_match"`, `view.slot_type`, `view.slot_count`, `view.elapsed_ms`
3. After gap detection summary: "Gaps detected: %d total (%s)" with extra `view.operation="gap_detection"`, `view.gap_count`, `view.elapsed_ms`
4. After ranking applied: "Ranking applied: method=%s" with extra `view.operation="ranking"`, `view.ranking_method`, `view.elapsed_ms`
5. At assembly complete: "Assembly complete: %d slots, %d gaps, %d edges" with extra `view.operation="assembly_complete"`, `view.total_slots`, `view.total_gaps`, `view.edge_count`, `view.elapsed_ms` (total time)

DEBUG log points:
1. Individual pattern match attempts: "Matching pattern '%s' against %s" with extra `view.operation="pattern_match_attempt"`, `view.pattern`, `view.slot_type`
2. Individual gap decisions: "Gap created for %s: %s" with extra `view.operation="gap_decision"`, `view.slot_type`, `view.severity`
3. Ranking score computations: "Density score for %s: %d" with extra `view.operation="density_score"`, `view.slot_id`, `view.score`

Use `logger.isEnabledFor(logging.DEBUG)` guard before DEBUG loops to avoid string formatting overhead.
