Remove `templates/.gitkeep` if it exists.

3. Create `templates/d2-structural.j2` that reproduces EXACTLY the output of the current `generate_d2()` function (lines 53-165). Use `-%}` trim blocks to control whitespace. Template receives context dict with keys: `spec_name`, `snapshot_id`, `sections`, `edges`, `gaps`, `abstraction_level`. Use custom filters `|sanitize_id` and `|gap_color`. 6. In `scripts/diagram_generator.py`, add:

   a. `import jinja2`, `import time`, `import logging`, `import os`, `import json` at top. Add `logger = logging.getLogger(__name__)`.

   b. New function `_build_template_context(view: dict, abstraction_level: str = "component") -> dict` that builds the flat context dict for templates. Pre-sorts all data for determinism (DIAG-08): sections sorted by slot_type, slots within sections sorted by name, edges sorted by (source_id, target_id, relationship_type), gaps sorted by (slot_type, index). Also computes `node_count`, `edge_count`, `direction` (for Mermaid), `has_unlinked`, `gap_severities` as template convenience vars.

   c. New function `_load_template(template_name: str | None, fmt: str, diagram_type: str, workspace_root: str | None = None) -> jinja2.Template` that:
      - If `template_name` is provided, look for that exact `.j2` file
      - Otherwise auto-select from manifest using `fmt + diagram_type` match (e.g., d2 + structural -> d2-structural)
      - Resolution: first check `{workspace_root}/templates/{file}` for user override, then fall back to built-in `templates/{file}` in skill directory
      - Register custom Jinja2 filters on the Environment: `sanitize_id` (wraps `_sanitize_id()`), `gap_color` (wraps `_GAP_COLORS.get(val, _GAP_COLORS["info"])`), `truncate_label` (truncates to 50 chars with "..." suffix)
      - Raise ValueError if no matching template found

   d. Refactor `generate_d2(view)` and `generate_mermaid(view)` to use template rendering:
      - Build context via `_build_template_context(view)`
      - Load template via `_load_template(None, fmt, diagram_type)`
      - Render with `template.render(**context)`
      - Keep the same function signatures for backward compatibility
      - The hardcoded string-building logic is REPLACED by template rendering

   e. Update `generate_diagram()` to:
      - Accept optional `template_name` parameter (from `spec.get("diagram_template")`)
      - Pass `workspace_root` to `_load_template()` for user override resolution
      - Use `time.perf_counter()` to measure generation elapsed time
      - Add `generation_elapsed_ms` to the diagram slot content dict
      - Pass template_name through to the rendering path

7. Determinism (DIAG-08): All sorting happens in `_build_template_context()` BEFORE template rendering. Templates just iterate pre-sorted data. Header comments (lines starting with `#` or `%%` containing "Generated from snapshot") are the only non-deterministic elements (contain snapshot IDs that change with registry state, but are deterministic for same input).
