  <name>Task 1: Abstraction layer pre-processor and BUILTIN_SPECS</name>
  <files>scripts/diagram_generator.py, scripts/view_assembler.py</files>
  <action>
1. In `scripts/diagram_generator.py`, add function `_apply_abstraction_level(view: dict, level: str) -> dict`:
   - If `level == "component"` or level is None, return the view unchanged (deep copy for safety)
   - If `level == "system"`:
     a. Identify "component" sections. For each component slot, check if it has sub-components (look for slots whose name or slot_id indicates they are children — use a `parent_id` or hierarchical naming pattern). Since the current data model may not have explicit parent-child relationships, use a simpler heuristic: if there are component_interface edges, the components at the source/target ends are "top-level" if they have the most connections. However, per CONTEXT.md the simpler approach is:
        - At system level, keep only top-level component slots (those that are not referenced as sub-components in other slots' fields)
        - For each top-level component, add a count badge to its name: "{name} (N sub-components, M interfaces)" by counting related slots from other sections
     b. Aggregate edges: group edges by (parent_source, parent_target) pair. When multiple edges exist between the same pair, collapse to a single edge with relationship_type showing count: "implements (3)"
     c. Return a new view dict (never mutate original) with the collapsed sections and aggregated edges
   - This function is called in `_build_template_context()` BEFORE sorting, when `abstraction_level` is passed

2. Update `_build_template_context(view, abstraction_level="component")` to call `_apply_abstraction_level(view, abstraction_level)` as the first step before sorting.

3. Update `generate_diagram()` to read `abstraction_level` from the spec: `spec.get("abstraction_level", "component")` and pass it through to the template context building path.

4. Update `scripts/view_assembler.py` BUILTIN_SPECS:
   - `"system-overview"`: add `"abstraction_level": "system"`
   - `"component-detail"`: add `"abstraction_level": "component"`
   - Other specs: no change (they default to "component")
