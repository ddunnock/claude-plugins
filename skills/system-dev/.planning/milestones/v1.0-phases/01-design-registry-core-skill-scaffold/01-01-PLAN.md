  <action>
    1. Create JSON Schema files in schemas/ directory using Draft 2020-12 ($schema: "https://json-schema.org/draft/2020-12/schema"). Use the component.json example from RESEARCH.md as the template. Each schema must have:
       - $schema, $id (https://system-dev.local/schemas/{type}.json), title
       - required: ["slot_id", "slot_type", "name", "version", "created_at", "updated_at"]
       - slot_id: string with type-specific pattern (comp-, intf-, cntr-, rref-)
       - slot_type: const matching the type name
       - name: string, minLength 1, maxLength 200
       - description: string (optional)
       - version: integer, minimum 1
       - created_at, updated_at: string, format date-time
       - status: enum ["proposed", "approved", "rejected", "modified", "deprecated"]
       - extensions: object (free-form, per user decision)
       - additionalProperties: false

       Type-specific fields:
       - component.json: source_block (string), parent_requirements (array of strings), rationale (string)
       - interface.json: source_component (string), target_component (string), protocol (string), data_format (string), parent_requirements (array of strings)
       - contract.json: component_id (string), interface_id (string), obligations (array of objects), vv_method (enum: test/analysis/inspection/demonstration), parent_requirements (array of strings)
       - requirement-ref.json: upstream_id (string, the REQ-NNN id), upstream_source (string, path to source registry), category (string), priority (string), trace_links (array of strings)

    2. Create scripts/shared_io.py with these utilities (use RESEARCH.md patterns):
       - atomic_write(filepath: str, data: dict) -> None: Write JSON atomically via temp file in same directory + fsync + rename. Per user decision: write to .tmp, atomic rename, orphaned .tmp on crash.
       - atomic_write_text(filepath: str, text: str) -> None: Same pattern for non-JSON text (used by journal).
       - cleanup_orphaned_temps(directory: str) -> list[str]: Walk directory tree, remove *.tmp files, return list of cleaned paths.
       - validate_path(path: str, workspace_root: str) -> str: Resolve with os.path.realpath(), verify it starts with workspace_root (path traversal protection per SCAF-03). Raise ValueError on violation.
       - ensure_directory(path: str) -> None: Create directory and parents if not exists.
       - All functions must have docstrings with REQ-ID references where applicable.

    3. Create scripts/__init__.py (empty, makes scripts a package).

    4. Create scripts/init_workspace.py:
       - init_workspace(project_root: str) -> dict: Creates .system-dev/ directory structure:
         - .system-dev/registry/components/
         - .system-dev/registry/interfaces/
         - .system-dev/registry/contracts/
         - .system-dev/registry/requirement-refs/
         - .system-dev/journal.jsonl (empty file)
         - .system-dev/index.json (initial: {"schema_version": "1.0.0", "updated_at": "<now>", "slots": {}})
         - .system-dev/config.json (initial: {"schema_version": "1.0.0", "workspace_version": "0.1.0", "created_at": "<now>"})
       - Must check if .system-dev/ already exists; if so, return warning dict (do not overwrite)
       - Must call cleanup_orphaned_temps() on startup for crash recovery
       - Returns dict with status, paths created, any warnings
       - Uses shared_io.atomic_write for index.json and config.json
       - Uses shared_io.validate_path for all path operations

    5. Create tests/test_init_workspace.py:
       - test_init_creates_workspace: init in temp dir, verify all dirs and files exist

       - Use pytest with tmp_path fixture for isolation
