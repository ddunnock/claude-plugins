  <files>schema-validator/src/index.ts, schema-validator/src/server/tools.ts, schema-validator/src/server/resources.ts, schema-validator/src/formats/types.ts, schema-validator/tests/server/plugin.test.ts, schema-validator/tests/server/startup.test.ts</files>
  <action>
1. Create src/formats/types.ts -- shared types used by all plans:
   - Export `FormatHandler` interface: `{ extensions: string[]; parse(content: string): unknown; serialize(data: unknown): string; }`
   - Export `ParseResult` type: `{ data: unknown; errors?: ParseError[]; partial?: boolean; }`
   - Export `ParseError` type: `{ message: string; line?: number; column?: number; }`
   - Export `FormatError` class extending Error with: code (string), filePath (string), line/column (optional numbers), suggestedFix (optional string)
   - Export `ToolError` type: `{ error: string; message: string; filePath?: string; details?: unknown; }`

2. Create src/server/tools.ts -- export function `registerTools(server: McpServer)`:
   - Register working Phase 1 tools:
     - `sv_parse_file`: inputSchema { filePath: z.string(), format: z.enum(["json","yaml","xml","toml"]).optional() }. Handler body: return placeholder "Implementation in Plan 02" for now (format handlers not yet built). This tool will be completed by Plan 02.
     - `sv_detect_format`: inputSchema { filePath: z.string() }. Handler body: return placeholder for now.
   - Register Phase 2+ stub tools (all return isError: true with NOT_IMPLEMENTED error):
     - `sv_validate`: inputSchema { filePath: z.string(), schemaName: z.string() }
     - `sv_read`: inputSchema { filePath: z.string(), schemaName: z.string() }
     - `sv_write`: inputSchema { filePath: z.string(), schemaName: z.string(), data: z.unknown() }
     - `sv_patch`: inputSchema { filePath: z.string(), schemaName: z.string(), patch: z.record(z.unknown()) }
     - `sv_register_schema`: inputSchema { name: z.string(), schema: z.unknown() }
     - `sv_list_schemas`: inputSchema {} (empty)
     - `sv_heal`: inputSchema { filePath: z.string(), schemaName: z.string(), mode: z.enum(["auto","suggest"]) }
   - All stub tools return: `{ content: [{ type: "text", text: JSON.stringify({ error: "NOT_IMPLEMENTED", message: "... will be available in Phase N" }) }], isError: true }`
   - Use snake_case with sv_ prefix per existing MCP naming conventions
   - NEVER use console.log -- use console.error for any logging

3. Create src/server/resources.ts -- export function `registerResources(server: McpServer)`:
   - Register resource "supported-formats" at URI "schema-validator://formats" returning JSON listing formats and extensions
   - Register resource "registered-schemas" at URI "schema-validator://schemas" returning empty list (populated in Phase 2)

   - Create StdioServerTransport, connect server to transport
   - Log to stderr: "Schema Validator MCP running on stdio"
   - Wrap in main().catch() that logs to stderr and exits with code 1
