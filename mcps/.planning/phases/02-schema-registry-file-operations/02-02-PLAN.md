  <name>Task 1: Implement sv_read, sv_validate, sv_write, sv_patch tool handlers</name>

  <behavior>
    sv_read:
    - Read file + parse with format handler + validate with zodSchema.parse -> return {data, format, filePath}
    - Schema not found -> isError with SCHEMA_NOT_FOUND
    - File not found -> isError with FILE_NOT_FOUND
    - Validation failure -> isError with VALIDATION_ERROR and Zod issues array (path, code, message, expected, received)
    - Path traversal -> isError with PathValidationError code

    sv_write:
    - Validate data with zodSchema.parse -> serialize with format handler -> mkdir -p -> atomicWrite
    - Returns {written: true, filePath, format}
    - Schema not found -> isError SCHEMA_NOT_FOUND
    - Validation failure -> isError VALIDATION_ERROR (rejects BEFORE writing to disk)
    - Creates parent directories if missing
    - Path traversal -> isError

    sv_validate:
    - Read file + parse + zodSchema.safeParse -> return {valid: true} or {valid: false, errors: ZodIssue[]}
    - Lightweight: no data returned on success
    - Schema not found -> isError SCHEMA_NOT_FOUND
    - File not found -> isError FILE_NOT_FOUND

    sv_patch:
    - Read existing file + parse + jsonMergePatch(existing, patch) + zodSchema.parse(merged) + serialize + atomicWrite
    - Returns {patched: true, data: mergedResult, filePath, format}
    - Null values in patch delete fields (RFC 7386)
    - Validation after merge -> isError VALIDATION_ERROR if merged result invalid
    - File not found -> isError FILE_NOT_FOUND
    - Schema not found -> isError SCHEMA_NOT_FOUND
  </behavior>

       - Return {data, format, filePath}

       **sv_write handler:**
       - validatePath(filePath)
       - registry.get(schemaName) -- if undefined, return isError SCHEMA_NOT_FOUND

       **sv_validate handler:**
       - validatePath(filePath)

       - atomicWrite(safePath, serialized)
       - Return {patched: true, data: merged, filePath, format}

       ```
         if (err instanceof SchemaRegistryError) -> isError with err.code
         default -> isError INTERNAL_ERROR
       }
       ```
