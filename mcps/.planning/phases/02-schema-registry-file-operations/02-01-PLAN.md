  <behavior>
    Registry:
    - register(name, zodSchema, source) stores schema and get(name) retrieves it
    - register(name, ...) with existing name throws SchemaRegistryError (immutable)
    - list() returns array of {name, source, fieldCount, extends?} metadata
    - register with extends="parent/name" merges parent fields + new fields into one Zod schema
    - register with extends="nonexistent" throws SchemaRegistryError

    JSON Schema converter:
    - jsonSchemaToZod({type:"object", properties:{name:{type:"string"}}}) returns z.object with name field
    - jsonSchemaToZod with required array marks fields as required
    - jsonSchemaToZod with nested objects produces nested z.object
    - jsonSchemaToZod with arrays produces z.array

    Merge patch (RFC 7386):
    - jsonMergePatch({a:1,b:2}, {b:3}) => {a:1,b:3}
    - jsonMergePatch({a:1,b:2}, {b:null}) => {a:1} (null deletes)
    - jsonMergePatch({a:{x:1}}, {a:{y:2}}) => {a:{x:1,y:2}} (deep merge)
    - jsonMergePatch({a:[1,2]}, {a:[3]}) => {a:[3]} (arrays replaced wholesale)
    - jsonMergePatch("old", {a:1}) => {a:1} (non-object target replaced)

    Discovery:
    - discoverSchemas scans skillPaths for schemas/*.ts files
    - Each file's named exports that are ZodType instances get registered as skillName/exportName
    - Invalid files (syntax errors, no Zod exports) are skipped with warning, not thrown
    - Uses validateSchemaFile from security/schema-loader.ts for pre-flight checks
  </behavior>
  <action>
    1. Install `zod-from-json-schema` dependency: `cd schema-validator && bun add zod-from-json-schema`

    2. Create `src/schemas/types.ts`:
       - `RegisteredSchema` interface: {name, zodSchema: ZodTypeAny, source: "discovered"|"registered", fieldCount, extends?: string}
       - `SchemaRegistryConfig` interface: {skillPaths: string[]}
       - `SchemaRegistryError` class extending Error with code field

    3. Create `src/schemas/json-schema-converter.ts`:
       - Export `jsonSchemaToZod(jsonSchema: Record<string, unknown>): ZodTypeAny`
       - Use `zod-from-json-schema` library for conversion
       - Wrap library errors in SchemaRegistryError with code "INVALID_JSON_SCHEMA"

    4. Create `src/schemas/merge-patch.ts`:
       - Export `jsonMergePatch(target: unknown, patch: unknown): unknown`
       - Custom ~20 line implementation per RESEARCH.md RFC 7386 spec
       - Non-object patch replaces target, null values delete keys, arrays replaced wholesale, objects merge recursively

    5. Create `src/schemas/registry.ts`:
       - Export `SchemaRegistry` class with:
         - `register(name: string, zodSchema: ZodTypeAny, source: "discovered"|"registered", extendsName?: string): void`
         - `get(name: string): RegisteredSchema | undefined`
         - `list(): Array<{name, source, fieldCount, extends?}>`
         - `registerFromJsonSchema(name: string, jsonSchema: Record<string, unknown>, extendsName?: string): void`
           - If extendsName provided: look up parent, use jsonSchemaToZod on new schema, merge with parent via z.object.merge()
           - Convert JSON Schema to Zod via jsonSchemaToZod, then call register()
         - `async discover(config: SchemaRegistryConfig): Promise<void>` -- delegates to discoverSchemas
       - Internal: `Map<string, RegisteredSchema>`
       - Throw SchemaRegistryError on duplicate name (code: "SCHEMA_EXISTS")
       - Throw SchemaRegistryError on missing parent for extends (code: "SCHEMA_NOT_FOUND")

    6. Create `src/schemas/discovery.ts`:
       - Export `async discoverSchemas(skillPaths: string[], registry: SchemaRegistry): Promise<void>`
       - For each skillPath: resolve absolute, scan schemas/*.ts files
       - For each file: call validateSchemaFile (SEC-03 pre-flight), then dynamic import with `await import(path.resolve(...))`
       - Extract named exports that are `instanceof z.ZodType`
       - Register as `skillDirName/exportName` with source "discovered"
       - On any error per file: console.error warning, skip, continue (per user decision)

    7. Write tests: registry.test.ts (register, get, list, duplicate, extends, missing parent), discovery.test.ts (scan, skip invalid, namespace), merge-patch.test.ts (all RFC 7386 cases)
  </action>

  </done>
