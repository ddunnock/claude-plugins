**Step 1: Replace sv_heal stub in tools.ts**

Add import at top of tools.ts: `import { healData } from "../schemas/healer.ts";`

Replace the sv_heal registration's `async () => notImplemented(3, "Self-healing")` callback with a real handler. The handler follows the established tool pattern:

1. `validatePath(filePath)` -- path safety
2. Check file exists with `Bun.file(safePath).exists()` -- return FILE_NOT_FOUND error if missing
3. Read content with `file.text()`, get handler via `getHandler(filePath)`, parse with `handler.parse(content)`
4. Get schema from `registry.get(schemaName)` -- return SCHEMA_NOT_FOUND error if missing
5. Call `healData(data, entry.zodSchema)` to get `HealResult`
6. If `mode === 'auto'` and `result.applied.length > 0`:
   - Serialize `result.data` with `handler.serialize(result.data)`
   - Write with `atomicWrite(safePath, serialized)`
   - Detect format from extension (same pattern as sv_read)
   - Return: `{ healed: result.healed, data: result.data, applied: result.applied, remaining: result.remaining, filePath: safePath, format: detectedFormat }`
7. If `mode === 'auto'` and `result.applied.length === 0` (already valid or all manual):
   - Do NOT write to disk
   - Return: `{ healed: result.healed, data: result.data, applied: [], remaining: result.remaining, filePath: safePath, format: detectedFormat }`
8. If `mode === 'suggest'`:
   - Do NOT write to disk
   - Return: `{ suggestions: result.applied, remaining: result.remaining }`
   - If already valid: `{ suggestions: [], remaining: [] }`

Error handling: same catch blocks as sv_validate (PathValidationError, FormatError, SchemaRegistryError, generic). Use `isError: true` only for real errors (missing file, missing schema, path traversal), NOT for heal results (even partial heals are success responses).

Also update the comment at the top of tools.ts to mark sv_heal as a Phase 3 working tool.

**Step 2: Create integration tests in tests/server/tools-phase3.test.ts**

Follow existing test patterns from `tests/server/` directory. Tests should:

Test cases:
- **Auto mode, fixable issues:** Write a JSON file with wrong types + missing defaulted fields. Call sv_heal with mode "auto". Verify response has applied fixes, file on disk is updated, re-reading the file passes validation.
- **Auto mode, already valid:** Write a valid JSON file. Call sv_heal with mode "auto". Verify response has empty applied/remaining, file on disk is unchanged.
- **Auto mode, partial heal:** Write a JSON file with both fixable and unfixable issues. Verify applied + remaining both populated, file on disk is written with partial fixes.
- **Suggest mode:** Write a malformed JSON file. Call sv_heal with mode "suggest". Verify response has suggestions array, file on disk is NOT modified (compare content before/after).
- **Suggest mode, already valid:** Write valid file. Verify `{ suggestions: [], remaining: [] }`.
- **Error: file not found:** Call with nonexistent path. Verify FILE_NOT_FOUND + isError.
- **Error: schema not found:** Call with unknown schema name. Verify SCHEMA_NOT_FOUND + isError.
- **Unknown fields preserved end-to-end:** Write JSON with extra fields not in schema + a fixable issue. Auto-heal. Re-read file. Verify extra fields still present.

Use `beforeAll`/`afterAll` for temp directory setup/teardown. Use `Bun.file().text()` to verify file contents.
