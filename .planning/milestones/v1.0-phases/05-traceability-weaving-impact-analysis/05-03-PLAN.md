    - Test: compute_impact with type_filter=["component"] returns only component nodes in paths (but traverses all types internally)
    - Test: compute_impact on a cyclic graph (A->B->C->A) terminates and returns correct results (visited set)
    - Test: compute_impact includes uncertainty_markers when graph has orphan nodes in traversed subgraph
    - Test: compute_impact returns tree-structured paths (hierarchical, not flat list)
    - Test: persist_impact(api, impact_result) creates impact-analysis slot in registry
    - Test: compute_impact on nonexistent start_id returns empty result with gap_marker
    - Test: format_impact_output produces hierarchical tree view string
  </behavior>
  <action>
    Add methods to the existing TraceabilityAgent class in scripts/traceability_agent.py:

    1. **compute_impact(start_id, direction="forward", depth_limit=None, type_filter=None)** -> dict:
       - Call self.build_or_refresh() to ensure fresh graph
       - If start_id not in graph nodes, return impact result with gap_marker indicating unknown element
       - Use BFS with collections.deque (per RESEARCH.md pattern, Pitfall 5 prevention with visited set):
         - For "forward": traverse graph["adjacency"]
         - For "backward": traverse graph["reverse_adjacency"]
         - For "both": run forward then backward, merge results
       - depth_limit: if not None, skip nodes at depth > depth_limit
       - Build tree-structured paths: each node in the result has children array containing its downstream impacts
         - During BFS, track parent for each discovered node
         - After BFS, reconstruct tree from root (start_id) using parent links
       - type_filter: if provided, only include nodes whose type is in the filter list in the OUTPUT paths, but still traverse all types during BFS (per CONTEXT.md: "full graph still traversed internally")
       - Calculate graph_coverage_percent: (nodes reachable from start_id / total nodes) * 100
       - If graph_coverage_percent < 100, add uncertainty_markers for unreachable subgraph areas (per CONTEXT.md)
       - Return dict matching impact-analysis schema structure

    2. **persist_impact(api, impact_result)** -> dict:
       - Create impact-analysis slot via api.create("impact-analysis", impact_result)
       - Return the create result with slot_id for reference

    3. **format_impact_output(impact_result)** -> str:
       - Tree view output per CONTEXT.md: hierarchical tree from changed element outward
       - Each level indented, showing element_id, element_type, and relationship
       - Example:
         ```
         comp-abc123 (component: Auth Service)
           |- intf-def456 (interface: Auth-Gateway) [boundary_of]
           |    |- cntr-ghi789 (contract: Auth Protocol) [constrained_by]
           |         |- vv:cntr-ghi789:OBL-01 (vv: Test Auth) [verified_by]
           |- cntr-jkl012 (contract: Auth Behavior) [constrained_by]
         ```
       - Show affected_count summary at bottom
       - Show uncertainty markers if present
       - If type_filter was used, note which types were filtered

    Add tests to tests/test_traceability_agent.py for each behavior. The test fixture from Plan 02 (small design graph with 2 needs, 3 reqs, 2 components, 1 interface, 1 contract) should be extended with a cycle edge to test cycle safety.
