**Slot ID patterns for ingested slots:** Ingested slots use `type:upstream-id` convention (e.g., `need:NEED-001`). The slot_id pattern in schemas must allow colons: `"pattern": "^need:.+$"`.

**schemas/need.json:**
- Required: slot_id, slot_type, version, created_at, updated_at, upstream_id, description
- Optional: stakeholder, source_block, source_artifacts, concept_dev_refs, upstream_status, rationale, upstream_registered_at, provenance, gap_markers, extensions
- slot_type const: "need"
- slot_id pattern: `^need:.+$`
- provenance: object with properties {source: string, upstream_file: string, ingested_at: string (format: date-time), hash: string}
- gap_markers: array of objects with {type: string (enum: missing_data, schema_mismatch, known_bug), finding_ref: string, severity: string (enum: low, medium, high), description: string}

**schemas/requirement.json:**
- Required: slot_id, slot_type, version, created_at, updated_at, upstream_id, description
- Optional: requirement_type, priority, upstream_status, parent_need, source_block, decomposition_level, upstream_attributes (object), quality_checks (object), rationale, upstream_registered_at, upstream_baselined_at, derives_from (array of strings for cross-refs), provenance, gap_markers, extensions
- slot_type const: "requirement"
- slot_id pattern: `^requirement:.+$`

**schemas/source.json:**
- Required: slot_id, slot_type, version, created_at, updated_at, upstream_id, name
- Optional: url, source_type, confidence (enum: high, medium, low, ungrounded -- GAP-4), research_context, concept_dev_ref, upstream_registered_at, provenance, gap_markers, extensions
- slot_type const: "source"
- slot_id pattern: `^source:.+$`

**schemas/assumption.json:**
- Required: slot_id, slot_type, version, created_at, updated_at, upstream_id, description
- Optional: category, status, source_block, related_requirements (array of strings), rationale, upstream_registered_at, provenance, gap_markers, extensions
- slot_type const: "assumption"
- slot_id pattern: `^assumption:.+$`

**schemas/traceability-link.json:**
- Required: slot_id, slot_type, version, created_at, updated_at, link_type, from_id, to_id
- Optional: rationale, upstream_registered_at, provenance, gap_markers, extensions
- slot_type const: "traceability-link"
- slot_id pattern: `^trace:.+$`

Add `ingest()` method to SlotAPI class. This method:
1. Accepts `slot_id: str, slot_type: str, content: dict, agent_id: str = "ingestion-engine"` -- note `slot_id` is provided, NOT auto-generated
2. Sets system fields: content["slot_id"] = slot_id, content["slot_type"] = slot_type, version = 1, timestamps
3. Checks if slot already exists (via self._storage.read). If exists AND version > 1 (meaning it was manually edited), SKIP and return {"status": "conflict", "slot_id": slot_id} -- do not overwrite local edits
4. If exists with version 1 (still at ingested version), treat as update: increment version, preserve created_at
5. Validates via self._validator.validate_or_raise
6. Writes via self._storage.write
7. Does NOT write individual journal entries (caller will write a batch summary entry)
8. Returns {"status": "created"|"updated"|"conflict", "slot_id": slot_id, "version": N}

CRITICAL: Do NOT modify the existing create() method. Existing tests depend on auto-generated IDs. The ingest() method is a separate code path.

**map_upstream_entry(entry: dict, field_map: dict, slot_type: str, upstream_file: str) -> dict:**
1. Iterate field_map, for each upstream_key -> slot_key: if upstream_key exists in entry, set slot_key in result
2. For required fields missing in upstream (checked against schema required list), log a warning -- do NOT silently use .get() with empty default (anti-pattern from RESEARCH.md)
3. Add provenance metadata: `{source: "requirements-dev", upstream_file: upstream_file, ingested_at: <ISO timestamp>, hash: content_hash(entry)}`
4. Return the mapped dict (without system fields -- SlotAPI.ingest() adds those)

**content_hash(entry: dict, exclude_keys: set | None = None) -> str:**
Deterministic SHA-256 hash from RESEARCH.md code example. Exclude timestamps (registered_at, baselined_at, updated_at, created_at).
