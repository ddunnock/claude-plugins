**scripts/delta_detector.py:**

Implement delta detection as described in RESEARCH.md Pattern 2.

`DeltaReport` dataclass or dict structure:
```python
@dataclass
class DeltaReport:
    added: list[dict]      # [{slot_id, entry}, ...]
    modified: list[dict]   # [{slot_id, entry, old_hash, new_hash}, ...]
    removed: list[str]     # [slot_id, ...]
    unchanged: list[str]   # [slot_id, ...]
    conflicts: list[dict]  # [{slot_id, reason}, ...] -- local edits vs upstream changes
    status_changes: list[dict]  # [{slot_id, old_status, new_status}, ...] -- Pitfall 5
```

`load_manifest(manifest_path: str) -> dict`:
- Load from `.system-dev/ingestion-manifest.json`
- If file doesn't exist, return empty manifest (first-time ingestion)
- Manifest schema: `{schema_version, ingested_at, upstream_path, upstream_state: {gates_status, counts}, hashes: {slot_id: hash_str}}`

`save_manifest(manifest_path: str, manifest: dict) -> None`:
- Atomic write via shared_io.atomic_write

`compute_deltas(upstream_entries: list[dict], manifest_hashes: dict[str, str], id_field: str, slot_prefix: str, slot_api: SlotAPI) -> DeltaReport`:
- Implement the algorithm from RESEARCH.md code example
- Additionally check for conflicts: for each "modified" entry, read current slot via slot_api. If slot version > 1 (local edit happened), classify as conflict instead of modified
- CRITICAL (Pitfall 5): Run delta detection on the FULL upstream set BEFORE status filtering. For entries whose status changed to non-processable (withdrawn/deprecated), add to status_changes list. The caller filters AFTER delta computation.

class IngestResult:

`ingest_all(upstream_path: str, workspace_root: str, schemas_dir: str) -> IngestResult`:
Main entry point. Steps:
1. Check upstream_path exists (raise FileNotFoundError if not)
2. Check upstream gate status via `check_upstream_gates()`. If not all passed, add warnings but continue (per CONTEXT.md: "warn but ingest")
3. Load manifest from `.system-dev/ingestion-manifest.json`
4. Initialize SlotAPI with workspace_root and schemas_dir

5. **Process each upstream registry in dependency order:**
   - needs_registry.json (needs depend on nothing)
   - source_registry.json (sources depend on nothing)
   - assumptions_registry.json (assumptions depend on nothing)
   - requirements_registry.json (requirements reference needs)
   - traceability_matrix.json (links reference everything)

   For each registry:
   a. Load via `load_upstream_registry(upstream_path, filename, top_key)`
   b. Compute deltas against manifest (first ingestion = all "added")
   c. For "added" and "modified" entries:
      - Run delta detection FIRST on full set (Pitfall 5)
      - Filter: skip entries with status "draft" or "withdrawn" (REQ-030). Record skipped count.
      - Map via `map_upstream_entry()` using the appropriate field map
      - For requirements: add `derives_from` cross-references. Map `parent_need` to `["need:{parent_need}"]`
      - Ingest via `slot_api.ingest(slot_id, slot_type, mapped_content)`
   d. For "removed" entries: do NOT delete slots. Instead, add a gap marker of type "upstream_removed" to the slot metadata. If the slot has downstream design artifacts (check by querying slots that reference it), flag as "breaking" severity; otherwise flag as "info" severity.
   e. Collect gap markers for traceability links (GAP-8: no backward traceability to concept-dev)

6. **Inject global gap markers:**
   - For all ingested needs: add GAP-1, GAP-2 markers (research gaps, ungrounded claims not in upstream)
   - For all ingested sources: GAP-3 if no citation graph, GAP-5 for no concept-dev access
   - For all ingested assumptions: GAP-7 (no lifecycle tracking)
   - For all traceability links: GAP-8 (no backward traceability to concept-dev)
   Note: gap markers go in the slot's `gap_markers` array field, not as separate slots

7. **Write batch journal entry:**
   ```python
   journal.append(
       "ingestion-batch", "system", "ingest",
       0, 1, "ingestion-engine",
       f"Ingested {total} items from {upstream_path}: {created} created, {updated} updated, {conflicts} conflicts",
       None, {"upstream_path": upstream_path, "counts": {...}}
   )
   ```

8. **Save updated manifest** with new hashes for all successfully ingested entries

9. **Write delta report** to `.system-dev/delta-report.json`:
   ```json
   {
     "schema_version": "1.0.0",
     "generated_at": "...",
     "upstream_path": "...",
     "summary": {"added": N, "modified": N, "removed": N, "conflicts": N, "unchanged": N},
     "details": {
       "added": [{slot_id, slot_type}, ...],
       "modified": [{slot_id, slot_type, fields_changed: [...]}, ...],
       "removed": [{slot_id, severity: "breaking"|"info"}, ...],
       "conflicts": [{slot_id, reason}, ...],
       "status_changes": [{slot_id, old_status, new_status}, ...]
     }
   }
   ```

10. **Write compatibility report** to `.system-dev/compatibility-report.json`:
    ```json
    {
      "schema_version": "1.0.0",
      "generated_at": "...",
      "findings": {
        "GAP-1": {"description": "...", "severity": "medium", "affected_slots": N, "affected_types": ["need"]},
        ...
      },
      "total_gap_markers": N,
      "total_affected_slots": N
    }
    ```

11. **Print console summary:**
    ```
    Ingestion complete:
      Needs: 48 ingested (0 skipped)
      Requirements: 420 ingested (59 skipped - draft/withdrawn)
      Sources: 21 ingested
      Assumptions: 16 ingested
      Traceability links: 1069 ingested
      Gap markers: 15 applied (across 6 findings)
      Conflicts: 0
    ```

**Error handling:**
- Malformed JSON in a single registry: log error, skip that registry, continue with others (partial ingestion per XCUT-01)
- Individual entry mapping failure: log warning, skip entry, continue
- Schema validation failure on a mapped entry: log error with entry ID and validation errors, skip entry, continue
