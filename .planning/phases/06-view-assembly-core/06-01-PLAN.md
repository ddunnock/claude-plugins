  <action>
Create `scripts/view_assembler.py` with these functions:

1. `load_view_spec(spec_path: str, parameters: dict | None = None) -> dict` — Load a view-spec JSON file, validate against view-spec schema, resolve {variable} placeholders in scope patterns using provided parameters. Raise ValueError if required parameters are missing.

2. `match_scope_pattern(pattern: str, slot_type: str, snapshot: dict) -> list[dict]` — Match slots in the snapshot against a glob-like scope pattern. Pattern syntax: `<slot_type>:<name_glob>` where name_glob supports `*` (any chars) and `?` (single char) using Python's `fnmatch` module. The slot_type portion is used to select the slot list from the snapshot, then the name_glob filters by slot `name` field.

3. `capture_snapshot(api: SlotAPI) -> dict` — Read index.json once atomically (single _load_index() call on the storage engine), then read all slot files referenced in that index. Return `{"snapshot_id": "<uuid>", "captured_at": "<iso>", "slots_by_type": {type: [slot_dicts]}}`. Deep copy all data so the snapshot is immutable. Access registry ONLY through SlotAPI (XCUT-04) — use api.query() for each slot type found in the index rather than direct file reads.

4. `build_gap_indicator(scope_pattern: str, slot_type: str, reason: str, gap_rules: dict | None = None) -> dict` — Create a gap indicator dict with severity determined by gap_rules config. Default severity: "warning". Gap rules is a dict mapping slot_type patterns to severity levels. Include `suggestion` field with actionable text referencing existing commands (e.g., "Run /system-dev:decompose to create components").

5. `load_gap_rules(workspace_root: str) -> dict` — Load gap severity rules from `.system-dev/gap-rules.json` if it exists, otherwise return default rules. Config format: `{"rules": [{"slot_type": "component", "severity": "error"}, ...]}`.

Important: The view_assembler does NOT use SlotAPI.create/update/delete — it only reads (XCUT-04 for reads, VIEW-10 preservation). All slot access through api.query() and api.read(), never through direct file I/O.

Add unit tests to `tests/test_view_assembler.py` for all functions above. Tests should create slots via SlotAPI in fixtures, then test matching/snapshot/gap functions against them.
